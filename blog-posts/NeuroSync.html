[2025-09-02] AI Chatbot

My very own AI Chatbot

5 min read By Sid Tewari

We're surrounded by incredible Large Language Models (LLMs), but many feel just out of reach. They're behind paywalls, login screens, or complex developer setups. I wanted to build something simpler: a clean, intuitive chatbot that anyone could use for a quick conversation, a burst of creativity, or to answer a question—no account required.

But there was a non-negotiable condition: security. If users provided their own API key to power the experience, I had to guarantee it would be handled with extreme care, vanishing without a trace the moment they left.

So I built a client-side chatbot powered by the DeepSeek API, designed around one core principle: ephemeral and secure API key handling.

Design

This chatbot came out really well, I went for the classic Cyberpunk theme since I love the game and comics so much. Honestly it was really fun designing the chatbot so it fits in that perfect blend of futuristic Cyberpunk theme.

The Architecture: Why this is not a typical chatbot

Most apps need a backend server to securely store secrets like API keys. But I wanted to avoid the cost and complexity of running a server. This led to a crucial decision:

The entire conversation would happen in the user's browser.

Their API key is sent directly from their machine to DeepSeek's API, never touching my servers. This means:

  • No Logging: I have no database to store keys or conversations.
  • No Middleman: The API request doesn't get proxied through my server; it's a direct, secure connection between the user and DeepSeek.
  • Total Privacy: The conversation is truly between the user and the AI.

The user's key is stored only in their browser's temporary memory (session storage) and is wiped clean the moment they close the tab.

The Build: A peek under the hood

The tech stack is simple and powerful:

  • Frontend: HTML, CSS, and vanilla JavaScript.
  • API: DeepSeek's Chat Completions API.
  • Security: The browser's session storage.

The Javascript Enginer (The magic part)

This is where the logic lives. The key is to capture the API and handle conversations.


  async function callDeepSeekAPI(userMessage) {
    const startTime = Date.now();
    
    try {
        const typingIndicator = showTypingIndicator();
        
        conversationHistory.push({ role: "user", content: userMessage });
        
        const response = await fetch('https://api.deepseek.com/chat/completions', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${apiKey}`
            },
            body: JSON.stringify({
                model: "deepseek-chat",
                messages: conversationHistory,
                stream: false
            })
        });
        
        if (!response.ok) {
            throw new Error(`API error: ${response.status} ${response.statusText}`);
        }
        
        const data = await response.json();
        const responseTime = Date.now() - startTime;
        
        updateMetrics(responseTime);
        
        if (document.getElementById('typingIndicator')) {
            document.getElementById('typingIndicator').remove();
        }
        
        return data.choices[0].message.content;
    } catch (error) {
        console.error("Error calling DeepSeek API:", error);
        const responseTime = Date.now() - startTime;
        updateMetrics(responseTime);
        
        if (document.getElementById('typingIndicator')) {
            document.getElementById('typingIndicator').remove();
        }
        
        throw error;
    }

Challenges & Learning

    CORS

    CORS is a Feature, Not a Bug: Initially, the browser blocked my API calls due to CORS (Cross-Origin Resource Sharing) policies. This is actually a critical security feature of browsers! DeepSeek's API is correctly configured to allow requests from any origin (*), so this worked. If it hadn't, I would have needed a backend proxy.

    Managing Chat History

    Keeping the conversation history array in sync with the UI was crucial for context. The AI's ability to remember what was said earlier hinges entirely on this history being passed back with every request.

Conclusion

This project shows that powerful AI doesn't have to be locked behind complex systems. With a few lines of JavaScript and a thoughtful approach to security, we can create engaging, personal, and private AI experiences that run directly in the user's hands.
