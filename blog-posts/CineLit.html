How I Bult a Media Recommendation System

6 min read By Sid Tewari

We've all been there. You finish a profound, gritty sci-fi book like Project Hail Mary and your streaming service recommends a campy 80s space opera. Or you watch a melancholic, character-driven film and are immediately served a slapstick comedy. Traditional recommendation engines are great for surface-level similarities (same genre, same actor) but they often miss the feel of what we're truly seeking.

I wanted to build something smarter. A system that doesn't just look at what you watched, but why you loved it. What was the mood? The underlying theme? The complexity?

So, I built a conversational AI recommendation engine that asks you questions and uses the powerful DeepSeek API to understand the nuance of your desires. Here’s how I did it.

Inspiration

Credit goes where credit is due, I took heavy inspiration from Netflix for the structuring of the recommendation system. Furthermore, I also took references for design and website theme from the official Ferrari website.

The Criterias

Before writing a single line of code, I defined the dimensions of a media recommendation that go beyond the basics. My system prompts the user for:

  • Genre: The non-negotiable starting point (e.g., Sci-Fi, Romance, Noir).
  • Key People: Specific authors, directors, or actors you're in the mood for.
  • Mood: The emotional resonance, do you want something lighthearted, dark, thought-provoking, or thrilling?
  • Themes: The big ideas, redemption, loss, technological singularity, love.
  • Release Era: A specific decade or a preference for classic vs. modern.
  • Complexity: How much mental energy do you have? Easy and fun or complex and challenging?

The Tool: Why DeepSeek?

I needed an AI model that could excel in conversational understanding and text completion. The model had to be powerful enough to parse a user's unstructured, descriptive prompt and generate coherent, creative, and accurate recommendations.

DeepSeek's LLM (Large Language Model) is perfectly suited for this. It's incredibly adept at understanding context and intent within a conversation, making it ideal for simulating a natural, recommendation-seeking dialogue.

Here is a simple API key call for DeepSeek on Python:

{
    from openai import OpenAI

    client = OpenAI(api_key="", base_url="https://api.deepseek.com")

    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": "You are a helpful assistant"},
            {"role": "user", "content": "Hello"},
        ],
        stream=False
    )

    print(response.choices[0].message.content)
}

Challenges & Learning

    Prompt Engineering

    The initial responses were verbose and didn't always follow the structure. Iterating on the system prompt—making it more explicit and demanding a strict JSON format—was crucial.

    Handling Ambiguity

    Sometimes a user's request is too vague (e.g., "something funny"). The next version will include a follow-up question mechanism to clarify.

    Cost & Efficiency

    Each API call has a cost. For a production app, you'd need to implement caching (e.g., storing common query results) to manage expenses.

Conclusion

This project proved that the future of discovery isn't in silent algorithms tracking our clicks, but in conversation. By leveraging the power of LLMs like DeepSeek's, we can build interfaces that understand the human nuance behind our search for stories.

The code I've shared is just a starting point. Imagine integrating this with databases like IMDb or Goodreads, or adding a voice interface. The possibilities are endless.
