How I Built CarbonOcean to Visualize Maritime Emissions

4 min read By Sid Tewari

If you’ve ever received a package from across the globe, you've benefited from the maritime shipping industry, the backbone of global trade. But this convenience comes with an invisible cost: it's a massive contributor to global carbon emissions, often operating out of sight and out of mind for the average consumer.

The data to understand this impact exists, but it’s trapped in complex spreadsheets and daunting databases, accessible only to data scientists and specialists. Regular people lacking such experience or expertise also need a clearer view to make informed decisions.

I built CarbonOcean to change that. It’s an open-source, interactive web app that transforms raw maritime emissions data into clear, intuitive visualizations. My goal was simple: to make the environmental impact of shipping transparent, understandable, and actionable for everyone.

Data

The martime shipping emission data being used is done via an Excel/csv file and the AI model is trained on that data. If you would like to look at another year or have more accurate data, all that needs to be done is to simply replace the file being studied.

The Mission: From Raw Data to Clear Insight

The core problem CarbonOcean solves is accessibility. The questions we need to answer are straightforward, but the path to the answers is not, such as "How does shipping emissions differ between countries" or "Based on the data, what can we expect the trend of carbon emissions to look like in the future".

CarbonOcean provides an interface to answer these questions without writing a single line of code. It’s designed for everyone from researchers to regular people interested in shipping and carbon emissions.

The Toolbox: Why Streamlit was the Perfect Choice

I needed a framework that could go from a Python script to a shareable web app with minimal friction. Streamlit was the obvious choice because it:

  • Turns Data into Web Apps Instantly: It seamlessly integrates with the PyData stack (Pandas, NumPy, Plotly) I use for data processing and visualization.
  • Is Intuitively Interactive: With built-in widgets like sliders and dropdowns, users can explore data on the fly, creating a dynamic experience.
  • Requires No Front-End Code: I could focus 100% on the data logic and user experience without getting bogged down in HTML, CSS, or JavaScript.

Challenges and the Future of CarbonOcean

  • Data Acquisition: The biggest challenge is sourcing clean, comprehensive, and up-to-date public data. Future versions will aim to integrate live data feeds.
  • From Visualization to Action: The next step is to add features like "what-if" scenarios (e.g., "What if 20% of ships on this route adopted green ammonia?").

Conclusion

CarbonOcean is more than a dashboard; it’s a democratization of knowledge. By making maritime emissions data accessible and visually compelling, we can foster a more informed public discourse and hold stakeholders accountable.

The climate challenge is vast, but it can be broken down into understandable parts. Tools like CarbonOcean help us do that, charting a data-driven course toward a more sustainable future for global shipping.
