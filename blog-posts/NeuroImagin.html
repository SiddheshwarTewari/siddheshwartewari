Text to Image AI: My Journey with OpenAI

4 min read By Sid Tewari

There's a special kind of magic in typing a sentence like "a cyberpunk cat drinking coffee in a rainy neon alley" and watching an AI bring it to life. Generative AI for images, like OpenAI's DALL-E, has exploded in popularity, but I wanted to build my own personal gateway to that magic, a clean, simple interface that focuses purely on the joy of creation.

The goal was straightforward: a text box for a prompt, a button to generate, and a gallery to display the results. But the most critical requirement was security. If users provided their own OpenAI API key, it had to be a temporary, client-side secret, vanishing without a trace when they left the page.

The result is a sleek, secure, and satisfying image generator built with the DALL-E API.

Multimodal

In the process of creating a text to image AI generator, it made me admire multimodal AI models just so much more. The complexity and the thought process required for just one type of output model is immense, imagine multiplying that by several times for a multimodal AI.

The Design: Why DALL-E and Client Side App?

OpenAI's DALL-E is one of the most powerful and reliable text-to-image models available. Its API is well-documented and produces consistently high-quality results.

Just like with the chatbot, I chose a client-side architecture. This means:

  • No Backend Server: There's no server cost for me and no data liability.
  • Direct Communication: Your browser talks directly to OpenAI's API over HTTPS.
  • Ephemeral Key Storage: Your API key is stored only in your browser's temporary memory and is wiped when you close the tab.

This approach puts you in complete control of your API key and your usage.

The Main Build

The OpenAI call I used in Javascript was the same one as provided in their FAQ, helping me connect to their model effectively.


  connectButton.addEventListener('click', function() {
                apiKey = apiKeyInput.value.trim();
                
                if (!apiKey) {
                    showError("Please enter a valid OpenAI API key");
                    return;
                }
                
                if (!apiKey.startsWith('sk-')) {
                    showError("API key should start with 'sk-'");
                    return;
                }
                
                // Test the API key with a simple request
                connectButton.innerHTML = ' Testing Connection...';
                
                fetch('https://api.openai.com/v1/models', {
                    method: 'GET',
                    headers: {
                        'Authorization': `Bearer ${apiKey}`
                    }
                }))

Security and Important Considerations

    Client-Side Key Management

    The API key is sent directly from the user's browser to OpenAI. It is never stored on my server. This is the core of the security model.

    Image Quality and Usage

    It is crucial to inform users that they are using their own API credits. DALL-E generation costs money per image, and users need to be aware of this.

Conclusion

Building this image generator was incredibly rewarding. It demystifies the process of working with powerful AI APIs and puts creative power directly into the user's hands. The combination of a simple interface and a robust, secure architecture makes it a perfect example of how accessible generative AI has become.

The potential for customization is endless: you could add options for image size, style, add a download button, or even create prompt templates.
